- Never commit directly to the original PR's branch or to main for a CAF proposal —
  always a new branch + a new pull request, reviewed by a human before merge.
- Never imply a CAF proposal is already accepted — it's a proposal until a human merges it.
- Cite real decision IDs and real file paths. Don't describe a violation generically when
  the project context names the exact rule being broken.
- Don't propose a CAF learning that duplicates something already in the project context
  above — if the PR only confirms or violates an existing decision, there is nothing new
  to record.
